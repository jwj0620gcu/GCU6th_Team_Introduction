# PRD.md

## 1. 문서 정보

- 제품명: Team Kanban Backend API
- 문서 버전: v1.0 (POC)
- 문서 목적: 소규모 팀이 공유 칸반 보드를 API/CLI로 사용하기 위한 제품 요구사항 정의

---

## 2. 배경과 문제 정의

팀 과제 수행 시 업무 상태가 분산되어 공유와 추적이 어렵습니다.  
이미 구축된 Supabase 칸반 DB를 기반으로, 팀원이 동일한 데이터에 접근할 수 있는 백엔드 API가 필요합니다.

---

## 3. 목표

1. Supabase 칸반 데이터를 FastAPI로 안정적으로 조회/수정할 수 있어야 한다.
2. 소규모 팀(3~10명)이 로컬 환경에서 즉시 실행해 사용할 수 있어야 한다.
3. FastAPI 내장 Swagger(`/docs`)로 API를 즉시 탐색/테스트할 수 있어야 한다.

---

## 4. 비목표(POC 범위 밖)

1. 로그인/인증/권한(Role) 관리
2. 멀티 테넌시(다중 조직 분리)
3. 클라우드 프로덕션 배포 및 고가용성 운영
4. 결제/요금/사용량 과금 시스템

---

## 5. 타깃 사용자

- 1차 사용자: 개발자(백엔드/CLI를 직접 사용하는 팀원)
- 팀 규모: 소규모 팀
- 사용 환경: 로컬 개발 환경(Mac/Linux/Windows)

---

## 6. 핵심 사용자 시나리오

1. 팀원이 보드와 컬럼을 조회해 현재 업무 상태를 확인한다.
2. 팀원이 새로운 이슈(업무)를 생성하고 담당자/라벨/마일스톤을 지정한다.
3. 팀원이 업무 진행에 따라 이슈를 다른 컬럼으로 이동하고 상태를 갱신한다.
4. 팀원이 코멘트를 남겨 협업 내역을 공유한다.
5. 외부 GitHub 이벤트가 들어오면 웹훅으로 DB 동기화를 수행한다.

---

## 7. 기능 요구사항

| ID | 요구사항 | 우선순위 | 완료 기준 |
|---|---|---|---|
| FR-01 | 서버 상태 확인 API를 제공한다. | Must | `/health` 호출 시 정상 응답 |
| FR-02 | 보드 목록 조회 API를 제공한다. | Must | 보드 리스트를 반환 |
| FR-03 | 보드별 컬럼 목록 조회 API를 제공한다. | Must | `board_id` 기준 정렬된 컬럼 반환 |
| FR-04 | 보드별 이슈 목록 조회 API를 제공한다. | Must | 이슈 + 담당자 + 컬럼 정보 반환 |
| FR-05 | 이슈 생성 API를 제공한다. | Must | DB `issues`에 신규 레코드 저장 |
| FR-06 | 이슈 수정 API를 제공한다. | Must | 제목/상태/컬럼/담당자/마일스톤 등 수정 가능 |
| FR-07 | 이슈 삭제 API를 제공한다. | Should | 삭제 후 목록에서 제외 |
| FR-08 | 코멘트 생성/조회/삭제 API를 제공한다. | Must | 이슈별 코멘트 관리 가능 |
| FR-09 | 라벨 조회/생성 및 이슈-라벨 연결 API를 제공한다. | Should | `labels`, `issue_labels` 반영 |
| FR-10 | GitHub 웹훅 수신 API를 제공한다. | Must | 유효 이벤트를 RPC로 반영 |
| FR-11 | 모든 API는 FastAPI Swagger 문서에 노출된다. | Must | `/docs`에서 확인 가능 |

---

## 8. 비기능 요구사항

1. 실행 환경: 로컬 단일 인스턴스 실행
2. 응답 형식: JSON
3. 오류 처리: 표준 HTTP 상태 코드 + 에러 메시지(`detail`)
4. 문서화: Swagger 및 docs 문서(PRD/TSD/DATABASE) 유지
5. 보안: 인증/로그인은 미적용(POC), 비밀키는 `.env` 관리

---

## 9. 데이터 범위

본 POC는 기존 Supabase 스키마를 사용합니다.

- 핵심 테이블: `boards`, `board_columns`, `issues`, `users`, `comments`, `labels`, `issue_labels`, `github_webhook_deliveries`
- 상세 스키마: [DATABASE.md](/Users/jungwonjun/database_cocone_team/docs/DATABASE.md)

---

## 10. 성공 기준 (Acceptance Criteria)

1. 로컬에서 FastAPI 서버가 실행되고 `/docs`가 정상 노출된다.
2. 보드/컬럼/이슈 조회 API가 정상 동작한다.
3. 이슈 생성/수정/삭제 및 코멘트 관리가 DB에 반영된다.
4. 웹훅 API 호출 시 `process_github_webhook` RPC가 실행된다.
5. 팀원이 동일 DB를 통해 작업 상태를 공유할 수 있다.

---

## 11. 오픈 이슈

1. 인증/권한 모델은 운영 단계에서 별도 정의가 필요하다.
2. 대량 트래픽/성능 최적화는 현재 POC 범위 밖이다.
3. CLI 명세는 별도 문서(PRD_CLI/TSD_CLI)에서 확장한다.
